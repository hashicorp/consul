# Consul Session Renew

Command: `consul session renew`

Corresponding HTTP API Endpoint: [\[PUT\] /v1/session/renew/:uuid](/api-docs/session#renew-session)

The `session renew` command renews a session.

The following table shows the [ACL configurations](/api#authentication) required to use the `session renew` command. You cannot configure 
[blocking queries](/api-docs/features/blocking) or [agent caching](/api-docs/features/caching)
from the command line, but you can configure them from the corresponding HTTP endpoint.

| ACL Required    |
| --------------- |
| `session:write` |
