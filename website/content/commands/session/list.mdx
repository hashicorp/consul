# Consul Session List

Command: `consul session list`

Corresponding HTTP API Endpoint: [\[GET\] /v1/session/list](/api-docs/session#list-sessions), [\[GET\] /v1/session/node/:node](/api-docs/session#list-sessions-for-node).

The `session list` command lists all sessions.

The following table shows the [ACL configurations](/api#authentication) required to use the `session list` command. You cannot configure 
[blocking queries](/api-docs/features/blocking) or [agent caching](/api-docs/features/caching)
from the command line, but you can configure them from the corresponding HTTP endpoint.

| ACL Required   |
| -------------- |
| `session:read` |

#### Command Options

- `-node` - Indicates whether to list all sessions in the cluster or only for a
  given node.

- `-format={pretty|json}` - Command output format. The default value is `pretty`.
