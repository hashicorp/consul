# Reset Access Control List (ACL) system

Once you bootstrap the Access Control List (ACL) system, Consul API endpoints, and CLI commands require a valid token to be accessed. The bootstrap process creates a token with `global-management` policy associated and, to maintain control over your Consul datacenter, you need to be able to access to a token with the same policy.

If you lose access to a valid `global-management` token, to regain access to your Consul datacenter, you need to reset the ACL system.

The process demonstrated in this document is non-disruptive. You will create a new `global-management` token but all previous tokens will remain in place and agent and service configurations will not be affected.

## Find Consul cluster leader

ACL reset must be performed on Consul cluster leader so you need to identify the correct server before proceeding with the reset.

Use the [`/status/leader` API endpoint](/consul/api-docs/status#get-raft-leader) to retrieve the Consul cluster leader's IP address.

```shell-session
$ curl 127.0.0.1:8500/v1/status/leader
"172.20.0.7:8300"
```

In this example, the leader is at at IP `172.20.0.7`. The following commands need to be run on that server.

## Find reset index number

Next, you need to obtain the index number of the previous ACL bootstrap. 

<Tabs>
<Tab heading="CLI" group="cli">

Use the [`consul acl bootstrap` CLI command](/consul/commands/acl/bootstrap) for that.

```shell-session
$ consul acl bootstrap
Failed ACL bootstrapping: Unexpected response code: 403 (Permission denied: ACL bootstrap no longer allowed (reset index: 27))
```

</Tab>
<Tab heading="API" group="api">

Use the [`/acl/bootstrap` API endpoint](/consul/api-docs/acl#bootstrap-acls) for that.

```shell-session
$ curl --request PUT http://127.0.0.1:8500/v1/acl/bootstrap
Permission denied: ACL bootstrap no longer allowed (reset index: 27)
```

</Tab>
</Tabs>

In this example, ACL bootstrap event is at index `27`.

## Reset ACL system

Reset the ACL system by creating a file, `<consul-data-directory>/acl-bootstrap-reset` containing the reset index number, in this case `27`.

```shell-session
$ echo 27 >> <data-directory>/acl-bootstrap-reset
```

Finally, re-bootstrap the ACL system to obtain a new management token.

<Tabs>
<Tab heading="CLI" group="cli">

Use the [`consul acl bootstrap` CLI command](/consul/commands/acl/bootstrap) to bootstrap the ACL system.

```shell-session
$ consul acl bootstrap
```
