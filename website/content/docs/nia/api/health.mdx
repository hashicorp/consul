# Health

The `/health` endpoints provide information on the health of a running Consul-Terraform-Sync (CTS) instance.

## Get Health

This endpoint returns the health of CTS.

| Method | Path                | Produces           |
| ------ | ------------------- | ------------------ |
| `GET`  | `/health` | `application/json` |

#### Response Statuses

| Status | Reason                                               |
| ------ | ---------------------------------------------------- |
| 200    | CTS is healthy                                       |
| 503    | CTS is unhealthy                                     |

#### Response Fields

| Name         | Type   | Description                                                                        |
| ------------ | ------ | ---------------------------------------------------------------------------------- |
| `request_id` | string | The ID of the request. Used for auditing and debugging purposes.                   |

#### Example: Getting the health of CTS

Request:

```shell-session
$ curl --request GET \
  localhost:8558/v1/health
```

Response:

```json
{
  "request_id": "b7559ab0-5111-381b-367a-0dfb7e216d41"
}
```
